# Markdown Style Guide

## General Principles
- **Clarity First:** Use clear, descriptive headers.
- **Consistency:** Use consistent formatting for lists, bold text, and code blocks.
- **Organization:** Use hierarchical headers (H1, H2, H3) to structure long documents.

## Formatting Rules
- **Headers:** Use ATX-style headers (e.g., # Header). Leave one blank line before and after headers.
- **Lists:** Use hyphens (-) for unordered lists. Use period-style (1.) for ordered lists.
- **Code:** Always specify the language for code blocks (e.g., \\\python).
- **Links:** Use descriptive link text rather than raw URLs.
- **Emphasis:** Use **bold** for key terms and *italics* for emphasis.

## Project Specifics
- **File References:** Use backticks for filenames (e.g., \config.yaml\).
- **Commands:** Use code blocks for shell commands.
- **Architecture References:** Always link to the Unified System Architecture when discussing system changes.
